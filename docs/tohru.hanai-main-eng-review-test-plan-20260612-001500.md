## Affected Pages/Routes
No web UI. The surface is a Slack workspace (DM or app mention to the bot) and
the local daemon process. Test from a Slack client against a dev workspace.

## Key Interactions to Verify
- DM the bot a known request ("review the engine repo") — ack arrives within
  seconds AND names the chosen workflow + extracted args; result lands in the
  same thread.
- Send a vague request — bot asks one clarification question naming what it
  needs; replying in-thread completes dispatch without retyping.
- Send a request that triggers a tool-permission escalation — approval request
  appears in the same thread; "approve" resumes the run; "deny" is honored.
- Reply something other than approve/deny to a pending approval — bot nudges
  "still waiting for approve/deny" and does NOT treat it as a new request.
- Send a second request while one is running — "busy, queued" reply; queued
  request runs after the first completes, in order.
- Message the bot from a non-allowlisted Slack account — polite refusal, no
  triage call, no workflow run.

## Edge Cases
- Two parallel agents escalate simultaneously — approval questions appear one
  at a time (serialized), each answer routes to the correct agent.
- Triage extracts args that fail the workflow's schema — clarification question
  names the bad/missing field.
- Triage returns a workflow id not in the registry — treated as clarification,
  not a crash.
- Workflow throws / budget exhausted — error summary posted in-thread, never
  silence.
- Kill the daemon mid-run (launchd restarts it) — bot reconnects; in-memory
  queue loss is expected and documented; no zombie disconnected state.

## Critical Paths
- Full dispatch loop: Slack message → allowlist → triage → ack with echo →
  runWorkflow → threaded result.
- Escalation round-trip: agent tool call → agentbus broker → Slack approval
  post → human reply → broker decision → agent resumes.
- Engine regression: runWorkflow WITHOUT cwd option still resolves defer rules
  from process.cwd() (existing CLI behavior unchanged).
- Triage eval: seed golden set (15-20 cases) passes against the live prompt.
