# Doc Kinds Manifest

Kind definitions consumed by `kusara`. The CLI loads `${KUSARA_DOC_ROOT}/kinds.md` at startup (default `KUSARA_DOC_ROOT=docs`); the YAML block below is the source of truth.

Schema semantics (`implements` / `depends_on` / `related` / `provides` / `modules`): invoke the `kusara:refs-schema` skill.

## Adding a new kind

1. Add an entry to the YAML block.
2. Add `index.output` if a generated INDEX is wanted.
3. Run `kusara validate`.

## Manifest

```yaml
# Each entry is a kind definition.
#   name           — kind identifier used as the `kind:` field in front matter
#                    and as the prefix in `id:` (e.g., kind=fr → id=fr:01-...).
#   path_globs     — repo-relative globs the scanner walks. Omit for kinds
#                    declared only via `provides:` of another doc.
#   declared_via   — "generated" if this kind is written only by `kusara index`.
#   id_pattern     — informational; what an ID for this kind looks like.
#                    Not enforced by the validator.
#   index          — when set, `kusara index` writes a generated INDEX for this
#                    kind to index.output. The generated file carries
#                    frontmatter (kind=index, generated=true, id=index:<name>).
kinds:
  - name: fr
    # Per-feature narratives. `[0-9]*` matches the numbered NN-slug files and
    # excludes index.md (which carries its own kind: index frontmatter).
    path_globs: ["docs/fr/[0-9]*.md"]
    id_pattern: "fr:{slug}"
    index:
      output: docs/fr/index.md

  - name: index
    declared_via: generated
    id_pattern: "index:{kind}"
```

## Reserved kind: `index`

Files generated by `kusara index` carry `kind: index` and are never hand-edited. Their `provides:` (or `indexes_kind`) enumerates what they index over.

## Out of scope

- Dated point-in-time docs under `docs/` (design snapshots, eng-review test
  plans, `docs/superpowers/**` plans and specs) are deliberately outside the
  graph: they are archival records, not living docs of record. The `fr` glob is
  scoped to `docs/fr/` so they carry no `refs:`.
- No glob expansion beyond standard shell globbing.
- `id_pattern` is documentation, not enforcement; uniqueness is the only ID check.
- Kinds with no `path_globs` and no `declared_via` fail validation.
